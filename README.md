# amplitude-pm-skills

**Open-source AI skills for product managers — battle-tested by PMs at Amplitude.**

Product managers spend too much time on the mechanics of writing specs, synthesizing research, and drafting comms — and not enough time on the thinking behind them. These skills give AI the structure and context to handle the mechanical parts, so you can focus on the decisions that matter.

Each skill is a standalone prompt template designed to work with Claude and other LLMs. Pick the ones that fit your workflow, customize them, and make them yours.

---

## Skills

| Skill | What it does |
|-------|-------------|
| [Spec Writing](skills/spec-writing/SKILL.md) | Turn messy ideas into a clear, structured PRD |
| [Experiment Design](skills/experiment-design/SKILL.md) | Write a hypothesis, define success metrics, and plan a holdout strategy |
| [Experiment Readout](skills/experiment-readout/SKILL.md) | Summarize experiment results, call a winner, and draft a recommendation |
| [Stakeholder Comms](skills/stakeholder-comms/SKILL.md) | Draft updates, announcements, or async briefs |
| [Discovery Synthesis](skills/discovery-synthesis/SKILL.md) | Extract themes and insights from raw user interviews and feedback |

## How to Use

1. Browse the `/skills` folder and find a skill that fits your task.
2. Open the `SKILL.md` file and copy the prompt template.
3. Paste it into your LLM of choice, fill in the placeholders, and run it.
4. Adapt the output to your context — these are starting points, not finished products.

## Contributing

We'd love your help making these skills better. Here's how:

- **Improve an existing skill** — open a PR with changes to any `SKILL.md` file.
- **Add a new skill** — create a new folder under `/skills` with a `SKILL.md` following the existing format.
- **Share feedback** — open an issue describing what worked, what didn't, or what's missing.
